# Out Of Scope

초기 버전에서 명시적으로 제외하거나 후순위로 미루는 항목을 정리한다.

---

## 현재 제외 범위

1. 오프라인 POS 실시간 연동
   - 초기에는 운영 정책으로 재고/주문 충돌을 흡수한다.
2. 복잡한 쿠폰/적립금
   - 핵심 구매/예약 흐름 이후 후순위로 둔다.
3. 추천/리뷰/커뮤니티 기능
   - MVP 핵심 기능이 아니므로 제외한다.
4. 번들 결제 (상품 + 예약 동시 결제)
   - PRD §6에 정의되어 있으나 1차 배포에서 제외한다. 스키마(`orders.bundle_id nullable`)는 준비 완료.
5. 이메일/앱 푸시 알림 채널
   - PRD §7의 3·4순위 채널. 1차 배포는 카카오 알림톡 + SMS fallback만 운영한다.
6. PG 실 연동
   - `FakePaymentProvider`를 유지한다. CircuitBreaker/timeout 설정은 프로덕션 준비 완료.

---

## 운영 원칙

- 이 문서는 “하지 않는 것”을 기록하는 PRD 보조 문서다.
- 범위 내 기능으로 승격되면 `docs/PRD/0001_기준_스펙/spec.md` 또는 별도 차기 PRD로 흡수한다.
